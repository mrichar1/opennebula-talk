Highly Available Virtualisation

Matthew Richardson

March 2013

What do we want?

Availability

  • Guests should always be running
  • Migrations should be live
  • No scheduled downtime

Redundancy

  • Host failures shouldn't matter
  • Disk images should be replicated
  • Buildings can (sometimes) burn down

Flexibility

  • Load balancing, monitoring, scheduling
  • Easy scaleability
  • Automate as much as possible

How do we get it?

Virtualisation

Libvirt & qemu/KVM

  • Handles live migrations
  • Hugely configurable
  • Guest OS agnostic

Storage

(rbd)

  • Network block devices
  • Distributed & replicated data
  • Massively scaleable
  • qemu integration

Ceph in Action

Cluster Management

OpenNebula

  • Management Interface
  • Controls hosts, disks, networks, guests
  • libvirt & ceph integration

Automation

OpenNebula

  • Resource creation from templates
  • Failover monitoring: hosts & guests
  • Load-balancing & guest placement

LCFG

  • Component for resource creation (& deletion)
  • ONEDB backup and restore methods
  • External Inventory integration
